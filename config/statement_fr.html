Capacitated Vehicle Routing

Dans le Vehicle Routing Problem, vous devez servir un ensemble de clients avec une flotte de véhicules.
Dans cette variante, le Capacitated Vehicle Routing Problem, les véhicules ont une capacité limitée.
Vous devez trouver l'ensemble de tournées le plus court possible.

  Règles

  • Les véhicules partent du dépôt (index 0) et doivent y retourner.
  • Chaque client doit être visité exactement une fois.
  • Le nombre de tournées ne doit pas dépasser le nombre de véhicules
  • Pour chaque tournée, la somme des demandes des clients visités ne doit pas dépasser la capacité des véhicules.
  • La distance entre deux points est Euclidienne - dist(a, b) = sqrt((xa - xb)2 + (ya - yb)2)

  Entrée du jeu

Le programme doit d'abord lire les entrées données, puis produire une seule ligne représentant les tournées des véhicules.
Entrée

Ligne 1: un entier k, le nombre de véhicules

Ligne 2: un entier c, la capacité des véhicules

Ligne 3: un entier n, le nombre de clients (+1 pour le dépôt)

Prochaines n lignes: 4 integers pour chaque client/dépôt

  • index, le numéro du client (ou 0 pour le dépôt)
  • x, la première coordonnée du client ou dépôt
  • y, la deuxième coordonnée du client ou dépôt
  • demand, la demande du client. Le dépôt (index=0) a une demande de 0.
Sortie

Une seule ligne contenant les tournées séparées par un point-virgule

Chaque tournée doit être les indices des clients séparés par un espace.

Le dépôt (0) ne doit pas être inclus dans la sortie.

Contraintes
5 <= n <= 200

Le temps de réponse est limité à 20 secondes.

  Exemple

Pour l'entrée du premier cas de test :

5 4 10 -> k = 4, c = 10, n = 5
0 0 0 0 -> dépôt en (0,0) - pas de demande
1 0 10 3 -> client 1 en (0,10) demande=3
2 -10 10 3 -> client 2 en (-10,10) demande=3
3 0 -10 3 -> client 3 en (0,-10) demande=3
4 10 -10 3 -> client 4 en (10,-10) demande=3

Quelques exemples de sorties au format correct :
  • 1 2 3;4
    Le premier véhicule fait la tournée 0 -> 1 -> 2 -> 3 -> 0. Le deuxième véhicule fait la tournée 0 -> 4 -> 0.
    La distance est dist(0, 1) + dist(1, 2) + dist(2, 3) + dist(3, 0) + dist(0, 4) + dist(4, 0) = 10 + 10 + sqrt(500) + 10 + sqrt(200) + sqrt(200) ≈ 80.6.
  • 4 2 1 3
    Le premier véhicule fait la tournée 0 -> 4 -> 2 -> 1 -> 3 -> 0.
    Cette solution est invalide : la somme des demandes est 3 + 3 + 3 + 3 > c = 10.
  • 1;2 4;3 2
    Cette solution est invalide : le client 2 est visité deux fois.
  • 1;3 4
    Cette solution est invalide : le client 2 n'est pas visité.
  • 1 2;3 4
    Cette solution est valide et optimale.
    La distance est dist(0, 1) + dist(1, 2) + dist(2, 0) + dist(0, 3) + dist(3, 4) + dist(4, 0) = 10 + 10 + sqrt(200) + 10 + 10 + sqrt(200) ≈ 68.3.

Quelques conseils

  • Le VRP est un problème académique NP-Difficile. Trouver une solution optimale est incroyablement difficile, donc vous devriez utiliser des algorithmes d'approximation - c'est l'heure de sortir vos métaheuristiques préférées !
  • Avez-vous déjà résolu le Travelling Salesman (TSP ou Voyageur de Commerce) ? Si oui, peut-être que vous pouvez réutiliser votre code : chaque tournée de véhicule est essentiellement un petit TSP.

À propos des instances de test et de validation

  • Les instances de test proviennent de la CVRPLib (sauf les 4 premières), spécifiquement des ensembles A (Augerat, 1995) et M (Christofides et al. 1979). N'hésitez pas à comparer vos résultats de test avec les optima connus.
  • Les tests fournis et les validateurs utilisés pour le calcul du score sont similaires mais différents.
